# Roles

Clarix implements a **15-role RBAC hierarchy** designed for FDA-regulated 503B compounding facilities. Each role maps to a specific persona in a cleanroom operation.

## Role Hierarchy

| Tier | Roles | Access Level |
|:----:|-------|-------------|
| **0** | `admin` | Super admin ‚Äî full system access |
| **1** | `pharmacist_in_charge`, `pharmacist`, `production_manager` | Management ‚Äî formula creation, batch release, team oversight |
| **2** | `qa_manager`, `qa_specialist`, `compounding_supervisor`, `procurement_manager`, `training_coordinator`, `executive` | Specialist ‚Äî domain-specific management |
| **3** | `compounding_technician`, `qc_technician`, `warehouse_clerk`, `maintenance_technician` | Operations ‚Äî execution and data entry |
| **4** | `read_only` | View only ‚Äî no data modification |

## Role Definitions

### Tier 0 ‚Äî System

**Admin** ‚Äî Full system access including user management, org settings, audit trail, and all module data. Cannot be assigned via self-service.

### Tier 1 ‚Äî Management

**Pharmacist in Charge (PIC)** ‚Äî Responsible for formula approval, batch release, and regulatory compliance. The PIC is the legally responsible individual per FDA regulations.

**Pharmacist** ‚Äî Creates and verifies formulas, reviews batch records, co-signs with PIC for release.

**Production Manager** ‚Äî Schedules batches, assigns technicians, manages production workflow.

### Tier 2 ‚Äî Specialist

**QA Manager** ‚Äî Manages deviations, CAPAs, environmental monitoring, and quality system oversight.

**QA Specialist** ‚Äî Assists QA Manager with deviation investigations and document control.

**Compounding Supervisor** ‚Äî Oversees technicians on the production floor.

**Procurement Manager** ‚Äî Manages vendor relationships, purchase orders, and supplier qualifications.

**Training Coordinator** ‚Äî Manages personnel qualifications, training records, and compliance tracking.

**Executive (VP)** ‚Äî Read-only executive dashboard with KPIs and audit readiness metrics.

### Tier 3 ‚Äî Operations

**Compounding Technician** ‚Äî Executes batch records step-by-step on iPad in the cleanroom.

**QC Technician** ‚Äî Performs quality control testing, records lab sample results.

**Warehouse Clerk** ‚Äî Manages receiving, inventory transactions, and stock levels.

**Maintenance Technician** ‚Äî Handles equipment calibration, cleaning logs, and room maintenance.

### Tier 4 ‚Äî View Only

**Read Only** ‚Äî Can view all data the is accessible to their organization. Cannot modify any records.

## Permission Rules

1. **Org Isolation** ‚Äî Every query is scoped by `organization_id`
2. **Separation of Duties** ‚Äî Batch execute ‚âÝ review ‚âÝ release (minimum 2 distinct users)
3. **Audit Trail** ‚Äî Every CREATE/UPDATE/DELETE writes to `audit_trail`
4. **E-Signature** ‚Äî Actions requiring signatures must re-authenticate
5. **Immutable Records** ‚Äî `audit_trail`, `e_signatures`, `inventory_transactions` are INSERT-only
