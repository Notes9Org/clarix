# Batch Records

The core module of Clarix — digital batch records replace paper-based manufacturing records with guided, step-by-step execution workflows.

## Workflow

```
Draft → Scheduled → In Progress → Pending Review → Pending Release → Released
```

## Key Features

- **Step-by-step execution** — Technicians follow guided workflows on iPad
- **Real-time tracking** — Production managers see live batch status
- **E-signatures** — Every critical step requires authenticated signatures
- **Barcode scanning** — Component verification via barcode scan
- **Tolerance checking** — Auto-flag out-of-spec measurements
- **Deviation linking** — Connect issues directly to batch records
- **BUD calculation** — Automatic beyond-use date based on formula

## Business Rules

| Rule | Description |
|------|------------|
| B1 | All required components must have `lot_status = 'released'` |
| B2 | Steps execute in order — step N+1 blocked until step N completes |
| B3 | Signature-required steps block progression until captured |
| B4 | Witness-required steps need a second user's signature |
| B6 | Out-of-tolerance values auto-flag and prompt deviation creation |
| B7 | Only QA Manager can transition `pending_review → pending_release` |
| B8 | Only PIC can transition `pending_release → released` |
